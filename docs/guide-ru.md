# Photo Metadata Extractor — Руководство пользователя

Инструмент для автоматического анализа вашей фотоколлекции: определяет, что изображено на каждой фотографии, извлекает доминирующие цвета и сохраняет всё в таблицу, которую можно открыть в Excel или Google Таблицах.

---

## Что нужно для работы

- **Docker Desktop** — бесплатная программа, которая запускает приложение в изолированной среде (Python и другие технические вещи устанавливать не нужно)
  - [Скачать для Windows](https://www.docker.com/products/docker-desktop/)
  - [Скачать для Mac](https://www.docker.com/products/docker-desktop/)
  - На Ubuntu/Debian Linux: `sudo apt install docker.io docker-compose-plugin`

- Около **500 МБ свободного места** на диске (для образа приложения и модели ИИ)
- Подключение к интернету при первом запуске (для загрузки модели, ~14 МБ)

1. Скачайте или склонируйте этот проект на компьютер
2. Всё — больше ничего устанавливать не нужно

## Первый запуск

### Windows

Дважды щёлкните **`start.bat`**, или нажмите правой кнопкой мыши на **`start.ps1`** → *Запустить с помощью PowerShell*.

### Linux / macOS

Откройте терминал в папке проекта и выполните:
```bash
./start.sh
```

### Что произойдёт дальше

Скрипт задаст два вопроса:

1. **Папка с фотографиями** — укажите путь к вашим фото (например `C:\Users\Анна\Картинки` или `/home/anna/Pictures`). Нажмите Enter, чтобы использовать значение по умолчанию.
2. **Порт** — просто нажмите Enter (по умолчанию 8080).

**При первом запуске** Docker собирает образ приложения — это занимает **2–5 минут**. Последующие запуски занимают несколько секунд.

Когда в терминале появится `Application startup complete`, откройте браузер и перейдите по адресу:

**http://localhost:8080**

## Использование веб-интерфейса

### Вкладка: Запуск

Здесь вы запускаете обработку фотографий.

**Конфигурация** (верхний блок) — отображает текущие настройки.

**Параметры запуска:**
- **Директория для сканирования** — нажмите на значок папки или на само поле, чтобы открыть браузер папок. Перейдите к нужной папке и нажмите **Выбрать**.
- **Кол-во цветов** — сколько доминирующих цветов извлекать с каждой фотографии (по умолчанию 5).
- **Пропустить обработанные** — отмечено по умолчанию. Обрабатываются только новые или изменённые фото. Снимите галочку, чтобы переанализировать всё заново.

Нажмите **Запустить**. Вы увидите:
- Прогресс-бар с количеством обработанных фотографий
- Живые логи в панели ниже

Скорость обработки на CPU: примерно **30–60 фотографий в секунду** для извлечения цветов. Классификация контента занимает немного больше.

---

### Вкладка: Фотографии

После обработки перейдите на вкладку **Фотографии**, чтобы увидеть результаты.

**Поиск** — введите имя файла или название категории (например `portrait`, `nature`, `food`), чтобы отфильтровать список.

**Столбцы таблицы:**
| Столбец | Описание |
|---------|----------|
| # | Внутренний ID |
| Название | Имя файла |
| Разрешение | Ширина × Высота в пикселях |
| Категории | Что обнаружил ИИ на фотографии |
| Цвета | Доминирующие цвета в виде кружочков |
| Обработано | Дата анализа |

**Нажмите на любую строку**, чтобы открыть детальную страницу фотографии в новой вкладке.

---

### Страница деталей фотографии

Показывает фотографию слева и все извлечённые метаданные справа:

- **Информация о файле** — название, разрешение, формат, путь, даты
- **Категории контента** — что обнаружил ИИ, с полосой уверенности. Чем длиннее полоса — тем увереннее определение
- **Доминирующие цвета** — цветовые образцы с названиями CSS-цветов, HEX-кодами и процентом площади изображения

## Понимание результатов

### Категории контента

Модель ИИ (MobileNetV2) обучена распознавать объекты из списка 1000 классов. Они сгруппированы в 11 категорий:

| Категория | Что определяет |
|-----------|----------------|
| `portrait` | Фотографии с человеческими лицами |
| `animal` | Собаки, кошки, птицы, рыбы, насекомые… |
| `nature` | Горы, озёра, леса, цветы, пляжи… |
| `food` | Пицца, фрукты, кофе, блюда с едой… |
| `vehicle` | Автомобили, мотоциклы, самолёты, лодки… |
| `architecture` | Церкви, замки, мосты, башни… |
| `city` | Улицы, светофоры, пешеходные переходы… |
| `indoor` | Мебель, мониторы, бытовая техника… |
| `sport` | Мячи, ракетки, лыжи, сёрфборды… |
| `document` | Книги, газеты, тетради… |
| `other` | Всё, что не попало в другие категории |

> **Важно:** Модель лучше всего работает с чётко видимыми объектами. Художественные фотографии, абстрактные фоны или необычные ракурсы могут дать низкую уверенность или категорию `other`.
>
> Определение портретов выполняется отдельным детектором лиц и, как правило, работает надёжнее для фотографий людей.

### Доминирующие цвета

Цвета извлекаются методом кластеризации K-Means. Для каждого цвета отображается:
- Цветовой образец
- Ближайшее **название CSS-цвета** (например «Darkolivegreen», «Sienna»)
- **HEX-код** (например `#4A7C2E`) — можно использовать в любом дизайнерском инструменте
- **Процент** от общей площади изображения

## Выходной файл

После обработки в вашей папке с фотографиями появится файл **`photo_metadata.csv`**. Его можно открыть в:
- Microsoft Excel
- Google Таблицах (Файл → Импорт)
- LibreOffice Calc

Каждая строка — одна фотография. Колонка `metadata` содержит JSON-строку с полными данными.

## Устранение неполадок

**Страница не открывается по адресу http://localhost:8080**
- Убедитесь, что Docker запущен
- Подождите ещё несколько секунд — приложение может ещё стартовать
- Проверьте, что порт 8080 не занят другим приложением (измените `PORT=8081` в файле `.env`)

**Ошибка «Docker is not running»**
- Откройте Docker Desktop и дождитесь, пока иконка кита на панели задач перестанет мигать

**Фотографии не отображаются в браузере папок**
- Браузер показывает только подпапки настроенного корня. Если фотографии находятся на другом диске (Windows), измените `BROWSE_ROOT` в файле `.env`

**Обработка идёт очень медленно**
- На CPU большие фотографии (20+ МП) обрабатываются дольше
- Папка из 1000 фотографий обычно обрабатывается за 30–60 секунд

**Категории определяются неверно**
- Модель может плохо справляться с необычными сюжетами — это известное ограничение ImageNet-классификаторов
- Для фотографий людей детектор лиц обычно работает надёжно
